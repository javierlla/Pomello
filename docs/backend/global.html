Middleware to check if a user is logged in by verifying the JWT token. Extracts the token from the 'Authorization' header, verifies its validity, and assigns the user's ID to the request object if the token is valid. If the token is missing or invalid, responds with a 401 status and an error message.

visualPomellodoroStatus(req, res) → {void}

Returns the current status of the Pomellodoro cycle, including the phase and cycle. The status contains six properties: - `running`: A boolean indicating whether the Pomellodoro cycle is currently active. - `timeouts`: The number of timeouts currently scheduled. - `sessions`: The number of active sessions (i.e. focus + break sessions) currently running. - `phase`: The current phase of the Pomellodoro cycle (either 'focus' or 'break'). - `cycle`: The current cycle of the Pomellodoro cycle (ranging from 1 to 4). - `startTime`: The timestamp (in milliseconds) when the Pomellodoro cycle started.
